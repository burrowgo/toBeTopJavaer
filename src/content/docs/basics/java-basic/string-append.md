---
title: "String Append"
---

In Java, the simplest way to concatenate strings is using the `+` operator.

Some people interpret the use of `+` for string concatenation in Java as operator overloading. In fact, it is not; Java does not support operator overloading. This is merely a feature of "syntactic sugar" provided by Java.

> **Operator Overloading:** In computer programming, operator overloading is a type of polymorphism. It involves redefining an existing operator and giving it another function to adapt to different data types.

> **Syntactic Sugar:** Syntactic sugar is a term coined by British computer scientist Peter Landin. It refers to syntax added to a computer language that does not affect the language's functionality but makes it more convenient for programmers to use. Syntactic sugar makes programs more concise and readable.

As mentioned earlier, using `+` for string concatenation is just syntactic sugar. Let's deconstruct this sugar and see how its internal principle is implemented.

Consider the following code:

```java
String wechat = "Hollis";
String introduce = "Daily updates on Java-related technical articles";
String hollis = wechat + "," + introduce;
```

Decompiling the generated bytecode using a tool like `jad` reveals the following:

```java
String wechat = "Hollis";
String introduce = "\u6BCF\u65E5\u66F4\u65B0Java\u76F8\u5173\u6280\u672F\u6587\u7AE0"; // Daily updates on Java-related technical articles
String hollis = (new StringBuilder()).append(wechat).append(",").append(introduce).toString();
```

By examining the decompiled code, we can see that during the concatenation process, the string constants are converted into a `StringBuilder`, and its `append` method is used.

Therefore, the implementation principle of `+` for string concatenation in Java is the use of `StringBuilder.append`.

However, not all string concatenations using `+` are based on `StringBuilder.append`. There is a special case: if two fixed literals are concatenated, such as:

```java
String s = "a" + "b";
```

The compiler performs "constant folding" (since both are compile-time constants known at compile time), directly turning it into `String s = "ab";`.
