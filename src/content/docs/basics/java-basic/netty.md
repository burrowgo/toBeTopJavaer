---
title: "Netty"
---

Netty is a non-blocking I/O client-server framework primarily used for developing Java network applications, such as protocol servers and clients. As an asynchronous event-driven network application framework and tooling, it simplifies network programming for tasks like TCP and UDP socket servers. Netty includes an implementation of the reactor programming pattern. Originally developed by JBoss, it is now developed and maintained by the Netty project community.

In addition to being an asynchronous network application framework, Netty includes support for HTTP, HTTP/2, DNS, and other protocols. It features the ability to run within Servlet containers, support for WebSockets, integration with Google Protocol Buffers, SSL/TLS support, and support for the SPDY protocol and message compression. Netty has been under active development since 2004.

Starting from version 4.0.0, Netty supports using NIO.2 as a backend while continuing to support NIO and blocking Java sockets.

**Essence:** A library (JAR package) created by JBoss.

**Purpose:** Rapid development of high-performance, high-reliability network server and client programs.

**Advantages:** Provides an asynchronous, event-driven network application framework and tools.
